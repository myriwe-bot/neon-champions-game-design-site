Story contract:
- Improve player-facing usability and targeting for the existing command pair only:
  - `rally_order` / `Rally Order`
  - `drone_strike` / `Drone Strike`
- Tactical command UI must communicate availability or denial for:
  - enough Command;
  - correct command profile;
  - Major Operation already used;
  - valid target / damaged active stack;
  - battle not complete.
- Drone Strike must require or clearly communicate the enemy target. Do not let the evidence rely on a hidden/unexplained deterministic fallback.
- Command summary and feedback must update immediately after valid and invalid attempts.
- Invalid command attempts must not mutate board state or Command state.

In scope:
- Minimal text/button UI affordances for existing Rally Order and Drone Strike.
- Target selection or selected-target feedback for Drone Strike.
- Visible denial/disabled reasons for invalid Rally/Drone attempts.
- PlayMode screenshots/evidence under `production/evidence/STORY-CMD-004/`.
- Regression tests for UI-visible state matching domain validation.
- Domain/application validation helper changes only if needed to support player-facing state; cover such changes with EditMode tests.

Out of scope:
- No new Champion command effects beyond Rally Order and Drone Strike.
- No full Operations/spellbook UI.
- No new Command economy rules, regeneration, cooldowns, round cadence, reactions, interrupts, initiative changes, or action economy redesign.
- No new Champion classes, skills, Doctrine/passives, levels, progression, loadouts, perks, equipment, or strategic AI command usage.
- No Intel integration, Field Upgrade replacement, operation upgrades, dirty information, fog/hidden information, save/load, campaign persistence, or final faction/canon changes.
- No final art/icons/portraits/VFX/audio/animation/final operation names/balance claims.
- Do not mark EPIC-005 DONE.

Acceptance criteria:
- Given a Marshal tactical battle, when the active stack is damaged, Rally Order is visibly available or clearly usable and using it spends Command and restores count.
- Given Rally Order is invalid, the UI shows a denial reason and no board/Command state mutation occurs.
- Given an Operator tactical battle, Drone Strike requires/communicates a valid enemy target and spends Command on success.
- Given Drone Strike was already used, a second attempt is visibly denied and Command does not change.
- Given command state changes, the tactical HUD command summary updates immediately.
- Given domain validation rejects a command, player-facing availability/denial state matches the domain reason instead of showing a contradictory affordance.
- Given this is a usability/targeting story, no new command mechanics, progression, final content, or broader Operations systems are added.
