Story contract:
- This is a narrow Champion encounter initiation clarity/affordance story.
- Preserve the human playtest complaint: Champions appear to be in the same area but battle does not clearly trigger; movement may be blocked; enemy interaction can feel like selecting the other Champion.
- Make the current existing Champion-vs-Champion encounter path understandable without adding new encounter mechanics.

In scope:
- Strategic snapshot/HUD/panel text/marker/button improvements for enemy Champion encounter state.
- Explicit player-facing affordance/control when the selected active Champion can launch an existing Champion-vs-Champion encounter.
- Denial/non-engageable text when encounter/movement is blocked by current existing validation.
- Friendly vs enemy Champion selection/inspection clarity so enemy UI cannot be mistaken for controlling the enemy Champion.
- Feedback for successful Champion encounter launch naming attacker, defender, source/location, and tactical Champion battle handoff.
- EditMode/PlayMode tests and evidence for these visible states.

Out of scope:
- No new strategic AI.
- No fog/ambush/stealth/hidden-information/diplomacy/zone-of-control/overwatch rules.
- No multi-Champion battles.
- No map redesign away from the current node graph.
- No objective/capture rule changes.
- No new battle mechanics, Command effects, units, attack types, balance, final art/icons/VFX/audio/animation/portraits/lore copy.
- No save/load or persistence expansion.
- No broad Champion movement/pathfinding rewrite.
- If a clarity fix requires gameplay rule changes, stop and report the exact rule-change need instead of expanding scope.

Acceptance criteria:
- Given the selected active Champion can launch an existing Champion-vs-Champion encounter, strategic UI shows an explicit Engage Champion/equivalent affordance naming the enemy Champion.
- Given an enemy Champion is present but not engageable, strategic UI explains why in player-facing terms instead of silently blocking movement or appearing to select the enemy.
- Given the player clicks/interacts with enemy Champion-related UI, feedback cannot be mistaken for taking control of the enemy Champion.
- Given Champion encounter launch succeeds, feedback names attacker, defender, source node/location, and that tactical Champion battle mode launched.
- Given Champion encounter launch is denied, feedback names the reason and does not mutate strategic/tactical state.
- Existing ChampionEncounterService battle setup/result mechanics remain unchanged unless the story records a stop-condition.
- Existing QA-006 state/action feedback remains visible and is not regressed.
