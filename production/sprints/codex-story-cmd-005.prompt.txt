Story contract:
- This is a narrow command explanation/readability pass for existing prototype commands only.
- Preserve the human playtest complaint: Rally Order and Drone Strike are visible but not understood; denial/result meaning is unclear; Marshal vs Operator identity is unclear.
- Explain existing mechanics without changing them.

In scope:
- Tactical HUD/panel/button/snapshot text explaining existing Rally Order and Drone Strike.
- Player-facing explanation of what each command does, Command cost, when it works, and why it may be denied.
- Marshal vs Operator prototype identity labels/descriptions in the current tactical UI.
- Compact display of Command state, cost, one-use/major-slot limits, target/result summary, and denial reason.
- EditMode/PlayMode tests and evidence for these visible explanations.

Out of scope:
- No new command effects beyond existing rally_order and drone_strike.
- No balance changes to costs, damage, healing, command pools, one-use limits, or profile stats.
- No new Command economy, cooldowns, regeneration, reactions, initiative, or action economy redesign.
- No full Operations/spellbook UI.
- No new Champion classes, skills, Doctrine/passives, levels, progression, loadouts, perks, equipment, or strategic AI command usage.
- No Intel integration, operation upgrades, dirty information, fog/hidden information, save/load, campaign persistence, objective/capture redesign, or final faction/canon changes.
- No final art, icons, portraits, VFX, audio, animation, final command names, or lore copy.
- No EPIC-005 closeout decision.
